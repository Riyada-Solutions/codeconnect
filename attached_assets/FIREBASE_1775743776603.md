# Firebase in this project

This document describes how **Firebase Core**, **Firebase Auth**, and **Firebase Cloud Messaging (FCM)** are wired into the Megasun / `base_app` Flutter app.

## Packages (`pubspec.yaml`)

| Package | Role |
|---------|------|
| `firebase_core` | App initialization |
| `firebase_auth` | Google / Apple sign-in flows (see auth feature) |
| `firebase_messaging` | Push notifications |
| `flutter_local_notifications` | System notifications (especially Android foreground) |

## Initialization

- **Entry point:** `lib/main.dart`
  - Calls `Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform)` after `WidgetsFlutterBinding.ensureInitialized()`.
- **Options:** `lib/firebase_options.dart` (FlutterFire CLI output). **Android** and **iOS** are configured; web/macOS/windows/linux throw if used without adding options.

Firebase is **not** initialized again inside `FcmHandler`; messaging assumes Core is already ready.

## FCM wrapper: `FcmHandler`

**File:** `lib/core/notification/fcm_handler.dart`

**Where it wraps the app:** `lib/features/common/presentation/modules/layout/layout_screen.dart` — the main shell wraps its child with `FcmHandler` and passes the app `NavigatorState` key so taps and dialogs can use context.

### Responsibilities

1. **`FirebaseMessaging.onBackgroundMessage`** — registered with `firebaseMessagingBackgroundHandler`. The handler body is currently **empty**; if you need data processing when the app is terminated/backgrounded, implement it there (must stay a top-level or static function, per FlutterFire rules).

2. **`flutter_local_notifications`**
   - Android default icon: `@drawable/ic_notification`
   - Creates channel `high_importance_channel` (“High Importance Notifications”)
   - **Note:** Foreground notifications shown via `NotificationUtils` use a different Android channel id (`2075FD` / “LocalNotificationChannel”) in `lib/core/notification/notification_utils.dart`.

3. **Permissions**
   - Android: notification permission via the local notifications plugin.
   - iOS: plugin `requestPermissions` + FCM foreground presentation options (`alert`, `badge`, `sound`).

4. **Streams / launch paths**
   - `getInitialMessage()` — cold start from notification; parses `NotificationPayload`, optional points dialog, then `setupNotificationClickAction`.
   - `FirebaseMessaging.onMessage` — foreground; shows local notification on **Android** with `message.notification` title/body; optional `pointsAdded` dialog.
   - `FirebaseMessaging.onMessageOpenedApp` — user opened from background; same dialog + `setupNotificationClickAction`.

5. **Token**
   - `_getToken()` logs the FCM token; on iOS it waits for APNS when needed.
   - `initialized` is set `true` after token attempt (including on error).

### Payload handling

- **`NotificationPayload`** — `lib/core/notification/data/notification_data_model.dart`  
  Built from FCM `data`: expects keys like `id` / `item_id`, `type`, optional nested JSON in `data` (string decoded to `Map`).

- **`NotificationType`** — `lib/core/constants/enums.dart`  
  Server `type` string must match the enum **value** (e.g. `points_added`, `product_created`, `rank_unlocked`, `redeem_product`).

- **Tap / deep link:** `setupNotificationClickAction` → `NotificationHelper.notificationNav` in `lib/core/notification/notification_type.dart`:

  | `NotificationType` | Navigation |
  |--------------------|------------|
  | `general` | No route |
  | `pointsAdded` | Points log screen |
  | `product` | Product details (`id` from payload) |
  | `rankUnlocked` | Layout with `currentPage: 2` |
  | `redeemProduct` | Layout with `currentPage: 2` |

- **`pointsAdded`:** Can show `showPaymentSuccessWithPointsDialog` when context is available; message text may come from `data.description` / `data.message` or notification body.

## Sending the token to your backend

- **`UpdateFCMTokenUseCase`** — `lib/features/auth/domain/usecase/update_fcm_token_usecase.dart`  
  Registered in `lib/features/auth/injection.dart`.

- **`LocalAuthProvider`** — `lib/features/auth/domain/provider/local_auth_provider.dart`  
  Calls `_updateFCMToken()` after a successful profile load when the user is logged in.

- **`getDeviceToken`** — `lib/core/notification/device_token.dart`  
  Caches token in module-level `kDeviceToken` and refreshes via `FirebaseMessaging.instance.getToken()`.

## Testing `points_added` notifications

See **`TEST_NOTIFICATION.md`** in the project root for example payloads and Firebase Console / cURL steps.

## Firebase Auth (Google Sign-In)

If Google Sign-In fails, the login flow may surface messages about **SHA-1** and Firebase Console configuration (`login_view_model.dart`). Ensure the Android app’s signing certificate is registered in the Firebase project.
